Introduction

[String] = "STONE"

It was towards the end of 2024 when i decided that i need yet another side project, specifically one i could finish over the course of a weekend. At that point of time, i was obsessed with language development and figured that a custom object notation and de-/serialization engine would fit the scope.

I always had this idea of an object notation with static typing, still no clue how this is supposed to be useful for anything that proper object notations didn't solve already, but i just gave it a shot. Additionally, i also wanted to experiment with a modular approach on tokenization and parsing, so this project was naturally a good fit and acted as a 2-in-1 learning experience.

So i grabbed one of my favorite languages (C#), spent my mental capacities on finding a really terrible name for my idea and ended up creating STONE. If it's not clear by the title of this post, the name is an abbreviation for Statically Typed Object Notation Experiment (i won't type that out another time, promised).

Implementation & Design

You might be thinking Hey, shouldn't the order be Design & Implementation and that would be a fair assumption, but due to one of the goals being a highly modular approach on tokenization and parsing this was actually one of the first things i tackled. The entire de-/serialization engine ended up being a bare 250 lines of code, would be even less if i'd follow the conventional and/or modern coding standards.

The engine is driven by two interfaces called Token Rule and Type Provider. Together with some recursive calls back into the engine, they handle most of the processing involved in serializing data and deserializing its STONE representation.

Here are the actual C# definition of those interfaces, starting with the one for Type Providers .. 

public interface IStoneTypeProvider
{
    bool CanProvideType(StoneEngineContext context, Type type);
    IStoneNode CreateNodeFromData(StoneEngineContext context, object data, Type type);
    IStoneNode CreateNodeFromSource(StoneEngineContext context, StoneTokenWalker tokenWalker, Type type);
    object CreateData(StoneEngineContext context, IStoneNode node, Type type);
    string CreateSource(StoneEngineContext context, IStoneNode node, Type type);
}

.. and here's the one for Token Rules: 

public interface IStoneTokenRule
{
    bool CanCreateToken(StoneSourceWalker sourceWalker);
    IStoneToken CreateToken(StoneSourceWalker sourceWalker);
